# النشر السحابي (تجربة أونلاين) + التنزيل على جهاز المسجد

نفس الكود يعمل في الحالتين. الفكرة: انشره أونلاين ليطّلع المسؤولون ويعدّلوا، ثم نزّله
على جهاز المسجد للتشغيل المحلي (دون إنترنت).

---

## أولاً: النشر أونلاين على Render (مجاني)

### 1) جهّز مستودع البيانات (لحفظ التعديلات)
الاستضافة المجانية بلا قرص دائم، فنحفظ البيانات في مستودع GitHub (نفس نمط «ناشر»):
- أنشئ مستودعاً **خاصاً** جديداً، مثل `masjid-data` (فارغ).
- أنشئ توكن: GitHub → Settings → Developer settings → **Fine-grained tokens** →
  صلاحية **Contents: Read and write** على مستودع `masjid-data` فقط. انسخه.

### 2) انشر على Render
1. ادخل render.com → سجّل بحساب GitHub.
2. **New → Blueprint** → اختر مستودع `masjid-screen`.
   (يقرأ `render.yaml` تلقائياً: build = `npm install`، start = `node core/standalone.js`.)
3. عند الطلب، أدخل متغيّرات البيئة السرّية:
   - `GITHUB_TOKEN` = التوكن الذي أنشأته.
   - `GITHUB_REPO` = `<حسابك>/masjid-data` (مثال: `drfajry/masjid-data`).
4. **Apply / Create** — ينشر خلال دقائق ويعطيك رابطاً مثل:
   `https://masjid-screen.onrender.com`

### 3) الروابط للمسؤولين
- **الشاشة:** `https://<اسم-الخدمة>.onrender.com/display/index.html`
- **لوحة التحكم:** `https://<اسم-الخدمة>.onrender.com/`  (دخول: admin / admin)

أرسل رابط اللوحة للمسؤولين ليعدّلوا الاسم/الموقع/الإقامة… وتُحفظ تعديلاتهم تلقائياً
على GitHub وتبقى رغم إعادة تشغيل الخدمة.

> ملاحظة الخطة المجانية: الخدمة «تنام» بعد ~15 دقيقة خمول، وأول فتح بعدها يأخذ ~30 ثانية
> ليستيقظ. عادي للتجربة والعرض.

---

## ثانياً: التنزيل على جهاز المسجد (تشغيل محلي دون إنترنت)

عند جاهزية الإعدادات، شغّله محلياً على جهاز المسجد:

1. ثبّت Node.js على الجهاز، وانسخ المشروع إليه.
2. (اختياري) لجلب نفس إعدادات المسؤولين التي حُفظت سحابياً: انسخ ملف `masjid-data.json`
   من مستودع البيانات إلى `data/masjid.json` داخل المشروع.
3. شغّل:
   ```bash
   npm install
   npm start          # وضع Electron بملء الشاشة (أو: npm run server للمتصفح)
   ```
4. يعمل **دون إنترنت** بالكامل. التحكم من الجوال عبر شبكة المسجد المحلية.

> محلياً لا تحتاج متغيّرات GitHub — التخزين يصير في ملف محلي تلقائياً.

---

## خلاصة الفرق
| | سحابي (Render) | محلي (جهاز المسجد) |
|---|---|---|
| الغرض | تجربة وتعديل المسؤولين عن بُعد | التشغيل الفعلي في المسجد |
| الإنترنت | مطلوب | غير مطلوب |
| التخزين | GitHub (تلقائي) | ملف محلي |
| التشغيل | `node core/standalone.js` | `npm start` |
