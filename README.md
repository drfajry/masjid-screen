# نظام شاشة مسجد ذكية 🕌

نظام عرض لشاشة المسجد يعمل على جهاز Mini PC، مع لوحة تحكم من الجوال أو المتصفح عبر الشبكة المحلية.
يعتمد على **تقويم أم القرى** ويعمل **بالكامل دون إنترنت**.

---

## المتطلبات

- **Node.js 18 أو أحدث** مثبّت على جهاز التشغيل.
- نظام التشغيل: Windows (موصى به للـ Mini PC) أو Linux.
- شاشة عرض + جهاز جوال على نفس الشبكة للتحكم.

---

## التثبيت

من داخل مجلد المشروع:

```bash
npm install
```

> التثبيت لا يحتاج أدوات بناء C++ — قاعدة البيانات تخزين JSON نقي (بدون مكتبات أصلية).
> يتطلب اتصالاً بالإنترنت **مرة واحدة فقط** عند التثبيت لتنزيل المكتبات. بعدها يعمل دون إنترنت.
> ميزة GPS/مراقبة المكسر عبر منفذ تسلسلي اختيارية وتحتاج مكتبة `serialport` (انظر قسمها).

---

## التشغيل

### الوضع الكامل (Electron — للجهاز الحقيقي)

```bash
npm start
```

- تفتح الشاشة مباشرة في وضع ملء الشاشة (Kiosk).
- يبدأ الخادم الداخلي تلقائياً ليستقبل لوحة التحكم.
- `Esc` للخروج من ملء الشاشة، `F11` للعودة إليه.

### وضع الخادم فقط (متصفح بدل Electron — اختياري)

مفيد للاختبار أو لتشغيل الشاشة عبر متصفح كامل الشاشة:

```bash
npm run server
```

ثم افتح في المتصفح:

- **الشاشة:** `http://<عنوان-الجهاز>:3777/display/index.html`
- **اللوحة:** `http://<عنوان-الجهاز>:3777/`

## لوحة التحكم من الجوال

1. تأكد أن الجوال على **نفس شبكة** جهاز العرض (واي‑فاي واحد).
2. اعرف عنوان IP للجهاز (يظهر في نافذة التشغيل، أو من إعدادات الشبكة).
3. افتح في متصفح الجوال:
   ```
   http://<عنوان-IP>:3777/
   ```
4. سجّل الدخول:
   - **المستخدم:** `admin`
   - **كلمة المرور:** `admin`
   - ⚠️ غيّر كلمة المرور من تبويب «المستخدمون» بعد أول دخول.

## التشغيل التلقائي عند الإقلاع

في وضع Electron يُفعّل النظام بدء التشغيل التلقائي مع المستخدم (`openAtLogin`).
للـ Mini PC الموصى به أيضاً:

1. فعّل الدخول التلقائي للمستخدم في Windows (تسجيل دخول بدون كلمة مرور).
2. ابنِ نسخة تثبيت عبر `npm run dist` وثبّتها، أو أضِف اختصار التطبيق إلى مجلد:
   ```
   shell:startup
   ```
3. بذلك يفتح النظام مباشرة في وضع العرض بعد أي انقطاع كهرباء.

## مزامنة الوقت

دقّة ساعة الجهاز مهمة لأوقات الصلاة. النظام يصحّح الفرق عبر **إزاحة** تُضاف
لساعة الجهاز عند العرض، دون تغيير ساعة النظام (لا يحتاج صلاحيات). من تبويب
«مزامنة الوقت» في اللوحة:

- **تلقائي:** يجلب الوقت الصحيح دورياً (افتراضياً كل ٦ ساعات) من مصدر موثوق
  عبر ترويسة `Date`، عند توفر الإنترنت. عند الانقطاع تبقى آخر إزاحة محفوظة.
- **يدوي:** أدخل الوقت الصحيح واضغط «ضبط الوقت» — مفيد للعمل دون إنترنت نهائياً.
- **معطّل:** استخدام ساعة الجهاز كما هي (تُصفّر الإزاحة).

يمكنك أيضاً الضغط على «مزامنة الآن» لمزامنة فورية، ومتابعة وقت آخر مزامنة
والإزاحة الحالية من نفس التبويب.

## جهاز GPS (اختياري)

عند توصيل جهاز GPS عبر USB يتعرّف النظام عليه تلقائياً ويضبط الموقع والوقت
دون إنترنت. أغلب الأجهزة تظهر كمنفذ تسلسلي وتبثّ جُمل NMEA فيها الإحداثيات
والوقت الدقيق.

الطريقة من تبويب «GPS» في اللوحة:

1. وصّل الجهاز، فعّل «تفعيل GPS»، ثم احفظ.
2. اضغط «بحث عن الجهاز تلقائياً» — يفحص النظام المنافذ ويختار ما يبثّ جُمل GPS
   صحيحة. (يمكن أيضاً اختيار منفذ محدّد بدل الكشف التلقائي.)
3. عند وصول إشارة (fix) يظهر عدد الأقمار والإحداثيات، ويُضبط الموقع تلقائياً
   (إن كان «ضبط الموقع تلقائياً» مفعّلاً)، والوقت من ساعة GPS الدقيقة (إن كان
   «ضبط الوقت تلقائياً» مفعّلاً).

**التثبيت:** ميزة GPS تعتمد مكتبة `serialport` المُدرجة ضمن
`optionalDependencies`. إن توفّرت عند `npm install` تعمل الميزة؛ وإن تعذّر
بناؤها يبقى النظام يعمل طبيعياً مع تعطيل GPS بهدوء. لتثبيتها/بنائها يدوياً:

```bash
npm install serialport @serialport/parser-readline
npx electron-rebuild -f -w serialport
```

> ملاحظة: المسجد ثابت، فأول إشارة جيدة تكفي لضبط الموقع. لا يُحدَّث الموقع إلا
> عند تحرّك ملموس (>30م) لتفادي الكتابة المتكررة.

## مراقبة إشارة المكسر (العودة التلقائية)

بعد الإقامة تعرض الشاشة وضع الصلاة، وعند انتهائها تعود للشاشة الرئيسية تلقائياً.
يعتمد ذلك على **إشارة المكسر الكهربائية** (نشط أثناء الصلاة، متوقف بعدها) — وليست
صوتاً يُحلَّل. عند بقاء الإشارة متوقفة للمدة المحددة (`مدة الانقطاع للعودة`) تعود الشاشة.

جهاز الـ Mini PC لا يقرأ فولت الكهرباء مباشرة، فيلزم وسيط بسيط يحوّل إشارة المكسر
إلى حالة رقمية. يدعم النظام طريقين (من تبويب «مراقبة المكسر»):

**1) عبر الشبكة (الأبسط والأكثر مرونة):** متحكّم صغير (Arduino/ESP) موصول بأوبتوكبلر
على إشارة المكسر، يرسل الحالة إلى نقطة في النظام:

```
POST http://<عنوان-الجهاز>:3777/api/signal
{ "active": true }     ← أثناء الصلاة
{ "active": false }    ← بعد انتهائها
```

أو إرسال نبضة متكررة أثناء الصلاة فقط (وتُعتبر متوقفة تلقائياً عند انقطاع النبض):

```
POST http://<عنوان-الجهاز>:3777/api/signal/ping
```

**2) عبر منفذ تسلسلي:** لوحة USB لدخل رقمي تظهر كمنفذ COM. يُحدَّد المنفذ، ويُستخرج
بِت الحالة من بايت الاستجابة عبر الإعدادات (موضع البايت/قناع البِت/البِت العالي =
نشط)، مع طلب hex دوري إن كانت اللوحة تتطلبه. (يعتمد هذا على مكتبة `serialport`
الاختيارية نفسها المستخدمة لـ GPS.)

تظهر الحالة حيّة في اللوحة (نشطة/متوقفة + آخر نشاط + المصدر). وفي حال تعطيل المراقبة
أو غياب الإشارة يعمل النظام بالمؤقّت الثابت كما هو.

## الثيمات (المظهر)

يدعم النظام عدّة مظاهر تُختار من تبويب «الهوية» في اللوحة، وتُطبَّق على الشاشة فوراً:

- **ليلي:** خلفية زرقاء داكنة بلمسات ذهبية (الافتراضي).
- **الجفالي:** خلفية عودية بهوية جامع الجفالي، خطوط بيضاء، وزخرفة هندسية فاتحة
  مع قوس محراب خلف الساعة.
- **زيتوني:** أخضر زيتوني مع لمسات ذهبية.
- **فاتح:** خلفية فاتحة بزخرفة رمادية، مع تمييز الأذان بالعودي والإقامة بالأخضر.

تقنياً: كل ثيم يُطبَّق عبر `data-theme` على `body`، ويعيد تعريف متغيّرات الألوان فقط
في `display/css/themes.css` دون تغيير البنية — فإضافة ثيم جديد تتم بكتلة متغيّرات واحدة.
شاشة الإقامة تبقى سوداء في كل الثيمات (متطلّب «أغلق الجوال»).

الخطوط: خط **Cairo** العربي مضمّن محلياً (`display/fonts/`) بأوزان ثقيلة وأحجام كبيرة
لوضوح القراءة للمصلّين من بعيد — ويعمل دون إنترنت.

لإكمال هوية الجفالي: ارفع شعار الجامع من «الهوية ← الشعار».

## التحديثات التلقائية

النظام يتحقق من التحديثات تلقائياً عند الإقلاع (عند توفّر إنترنت)، وتُدار أيضاً
يدوياً من تبويب «التحديثات» في اللوحة: التحقق، متابعة التنزيل، ثم «تثبيت وإعادة
التشغيل». تعمل هذه الميزة في **النسخة المثبّتة فقط** (المبنيّة عبر electron-builder)،
وتُعطَّل بهدوء في وضع التطوير/الخادم.

النشر عبر **GitHub Releases** (مجاني):

1. عدّل `build.publish` في `package.json` ليطابق مستودعك:
   ```json
   "publish": [{ "provider": "github", "owner": "<اسمك>", "repo": "<المستودع>" }]
   ```
2. ارفع رقم الإصدار في `package.json` (مثلاً `1.0.0` ← `1.0.1`).
3. ابنِ وانشر:
   ```bash
   set GH_TOKEN=رمز_github_الخاص   # توكن بصلاحية إنشاء Releases
   npm run dist -- --publish always
   ```
   هذا يبني المثبّت ويرفعه كإصدار على GitHub.
4. الأجهزة المثبّت عليها النظام ستكتشف الإصدار الجديد وتنزّله، ثم تُثبّته عند إعادة
   التشغيل (أو فوراً عبر زر «تثبيت وإعادة التشغيل» في اللوحة).

> لا تكلفة استضافة — GitHub Releases مجاني. ويبقى النظام يعمل دون إنترنت؛ التحديث
> اختياري عند توفّره فقط.

## العمل دون إنترنت

كل العمليات الأساسية تعمل دون إنترنت:

- حساب أوقات الصلاة (مكتبة `adhan` مضمّنة محلياً في `vendor/`).
- التاريخ الهجري (تقويم أم القرى عبر `Intl` المدمج في النظام).
- الإقامة، العدّاد، الأذكار، الجنازة، جميع الإعدادات، وقاعدة البيانات المحلية.

الإنترنت مطلوب فقط لـ: التثبيت الأول، والتحديثات (اختياري)، ومزامنة الوقت الدورية (اختياري).

عند إضافة وحدة جديدة، أضِف وسم `<script>` لها في `index.html` بالترتيب الصحيح.

## ملاحظات أمان

- النظام مخصّص للشبكة المحلية للمسجد. لا تعرّضه للإنترنت العام مباشرة.
- قراءة الإعدادات للعرض متاحة دون تسجيل دخول (الشاشة لا تملك حساباً)؛
  أمّا أي **تعديل** فيتطلب تسجيل دخول.
- غيّر كلمة مرور `admin` الافتراضية فور التشغيل.
