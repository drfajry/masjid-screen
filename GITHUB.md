# رفع المشروع على GitHub + نشر التحديثات

التحديثات التلقائية تعمل عبر **GitHub Releases**. الخطوات مرة واحدة، ثم كل تحديث = إصدار جديد.

## ١) إنشاء المستودع
أنشئ مستودعاً على GitHub باسم **masjid-screen** تحت حسابك (مطابق لإعداد `build.publish`
في `package.json`: `owner: drfajry`, `repo: masjid-screen`). لو اخترت اسماً مختلفاً، عدّل
`build.publish` ليطابقه.

> اجعل المستودع **عاماً (Public)** — أبسط طريقة لعمل التحديثات التلقائية مجاناً، ومناسب لعمل خيري.

## ٢) رفع الملفات أول مرة
من داخل مجلد المشروع:

```bash
git init
git add .
git commit -m "أول إصدار - نظام شاشة المسجد"
git branch -M main
git remote add origin https://github.com/drfajry/masjid-screen.git
git push -u origin main
```

> `.gitignore` يستثني `node_modules` و`dist` و`data` تلقائياً — لا تُرفع.

## ٣) رمز وصول (Token) لنشر الإصدارات
أنشئ توكن من: GitHub ← Settings ← Developer settings ← Personal access tokens
بصلاحية `repo` (إنشاء Releases). احفظه بأمان.

## ٤) نشر تحديث جديد
كل ما تريد إصدار تحديث:

1. ارفع رقم الإصدار في `package.json` (مثل `1.0.0` ← `1.0.1`).
2. احفظ التغييرات وارفعها:
   ```bash
   git add . && git commit -m "تحديث 1.0.1" && git push
   ```
3. ابنِ وانشر الإصدار:
   ```bash
   # ويندوز (PowerShell):
   $env:GH_TOKEN="ضع_التوكن_هنا"
   npm run dist -- --publish always
   ```
   هذا يبني المثبّت ويرفعه تلقائياً كإصدار على GitHub (مع ملف `latest.yml` الذي
   يحتاجه التحديث التلقائي).

## ٥) ماذا يحدث على أجهزة المساجد
- عند الإقلاع (وعند الضغط على «التحقق من التحديثات» في اللوحة) يكتشف النظام الإصدار الأحدث.
- يُنزّله في الخلفية، ثم يُثبّته عند إعادة التشغيل أو فوراً عبر زر «تثبيت وإعادة التشغيل».

## نقاط مهمة
- أول مرة فقط: ثبّت النظام على الجهاز يدوياً من المثبّت الذي بنيته (`dist/`).
  بعدها تتولّى التحديثات التلقائية الباقي.
- رقم الإصدار يجب أن يزيد في كل نشرة وإلا لن يُعتبر تحديثاً.
- المستودع العام لا يتطلب توكن على أجهزة المساجد (التنزيل عام)؛ التوكن مطلوب فقط
  على جهازك وقت النشر.
