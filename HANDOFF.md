## 🧩 البنية المعيارية (تقسيم الملفات)

تم تقسيم الملفات الكبيرة إلى وحدات صغيرة لتسهيل التعديل والرجوع:
- **الخادم**: `config-defaults.js` + `db.js` + `routes.js` + `timesync.js` + `nmea.js` + `gps.js` + `signal.js` + `cloud-store.js` (تخزين GitHub اختياري) + `server.js`، ونقطة التشغيل السحابي `standalone.js`، وفي الجذر `updater.js`. النشر السحابي عبر `render.yaml` + دليل `CLOUD.md`.
- **الشاشة** `display/js/`: core، clock، tone، render، countdown، adhkar-data، adhkar-view، state، connection — تتشارك الكائن العام `MD`. والثيمات في `display/css/themes.css`.
- **اللوحة** `panel/js/`: core، auth، forms، save، users-logs، timesync، gps، signal، update، links (روابط الوصول + QR)، boot — تتشارك الكائن العام `App`. أدلّة: `QUICKSTART.md` (التشغيل) و`GITHUB.md` (الرفع والتحديثات).
- وحدات المتصفح تُحمّل بوسوم `<script>` مرتّبة في `index.html` (core أولاً، connection/boot أخيراً).
- جميع الوحدات اجتازت اختبار التحميل (كل الدوال متّصلة، وكل أزرار اللوحة موجودة) واختبار الخادم end-to-end (12/12).

## ✅ المنجز والمختبَر

**محرك أوقات الصلاة (`core/prayer.js`)** — مُتحقّق منه برمجياً:
- تقويم أم القرى (زاوية الفجر 18.5°، فاصل العشاء 90 دقيقة) عبر مكتبة `adhan` المضمّنة محلياً.
- تعديل كل صلاة بالدقائق (+/−)، عشاء رمضان المستقل (يُطبّق تلقائياً في شهر 9 فقط).
- أوقات الإقامة، ثلث الليل الأخير (البداية + المتبقي)، الصلاة القادمة (مع الانتقال لفجر الغد بعد العشاء).
- التاريخ الهجري مع إضافة/إنقاص أيام (للرؤية الشرعية)، كشف رمضان.
- تم التحقق للمدينة المنورة: الفجر/المغرب/العشاء مع التعديلات، ثلث الليل، والتاريخ الهجري — كلها صحيحة.

**قاعدة البيانات (`core/db.js`)** — مُختبَرة end-to-end:
- تخزين JSON نقي: `config`، `users` (salt + sha256)، `logs` — بكتابة ذرّية، بدون مكتبات أصلية.
- دمج عميق للإعدادات يحافظ على المفاتيح القديمة عند الترقية، تسجيل تلقائي للتغييرات.
- مستخدم افتراضي `admin/admin`، إضافة/حذف/تحقق، سجل العمليات مع بحث وتصفية وتصدير CSV (مع BOM للعربية).

**الخادم (`core/server.js`)** — مُختبَر end-to-end:
- Express + WebSocket. جميع المسارات تعمل: تسجيل الدخول، الإعدادات، الشعار، الأوامر، المستخدمون، السجل، التصدير.
- البث الفوري عبر WebSocket: تحديث الإعدادات وأوامر الشاشة تصل للعرض لحظياً (تم التحقق).
- عقد الأوامر متّسق عبر لوحة التحكم ← الخادم ← الشاشة (`command` كنص: `start_iqama`/`show_adhkar`/`return_normal`/`reload` + `prayer`).

**مزامنة الوقت (`core/timesync.js` + `display/js/clock.js` + `panel/js/timesync.js`)** — مُختبَرة end-to-end:
- نظام إزاحة (offset) يصحّح فرق ساعة الجهاز دون تغيير ساعة النظام (لا يحتاج صلاحيات).
- ثلاثة أوضاع: تلقائي (يجلب الوقت الصحيح دورياً من ترويسة Date عند توفر الإنترنت، مع تصحيح نصف زمن الذهاب والإياب)، يدوي (المسؤول يضبط الوقت)، معطّل.
- مسارات `POST /api/time/sync` و`POST /api/time/manual`، وإعادة جدولة تلقائية عند تغيير الإعداد، وبث الإزاحة للشاشة فوراً.
- الشاشة تطبّق الإزاحة عبر `MD.now()`. تم التحقق من حساب الإزاحة (±ثانية)، والحفظ، والبث، ومعالجة فشل الاتصال بأمان.

**الثيمات + أيقونة التطبيق** — مُنجزة ومُعايَنة بصرياً:
- أربعة ثيمات عبر `data-theme` على `body` في `display/css/themes.css` (متغيّرات ألوان فقط، البنية ثابتة): ليلي (افتراضي)، **الجفالي** (عودي #862530 + خطوط بيضاء + زخرفة + قوس محراب)، زيتوني، فاتح (أذان عودي/إقامة خضراء).
- تُختار من تبويب «الهوية» (`display.theme`) وتُطبَّق فوراً عبر البث. شاشة الإقامة تبقى سوداء في كل الثيمات.
- عوينت أفقياً وعمودياً للثيمات الأربعة عبر Playwright — تطابق النماذج المرفقة.
- أيقونة التطبيق `assets/icon.ico` (متعددة المقاسات 16→256) بهوية الجفالي (عودي + مسجد أبيض + هلال + محراب)، مربوطة في `package.json` (build.win.icon + nsis).
- خط **Cairo** العربي مضمّن محلياً (`display/fonts/` + `display/css/fonts.css`) بأوزان 500/700/900، مع تكبير أحجام الساعة/المواقيت/الأسماء وزيادة الأوزان لوضوح القراءة من بعيد (أفقي وعمودي). يعمل دون إنترنت.

**التحديثات التلقائية (`updater.js` + مسارات + `panel/js/update.js`)** — مُختبَرة end-to-end:
- `electron-updater` عبر GitHub Releases (إعداد `build.publish` في package.json: owner=drfajry، repo=masjid-screen — يُعدّل للمستودع الفعلي).
- فحص تلقائي بعد الإقلاع بـ12 ثانية، وتحكّم يدوي من اللوحة: تحقّق/متابعة تنزيل/تثبيت وإعادة تشغيل.
- مسارات `GET /api/version` (بلا مصادقة)، `GET /api/update/status`، `POST /api/update/check`، `POST /api/update/install`.
- يعمل في النسخة المبنيّة فقط؛ وفي وضع standalone/التطوير يُعطَّل بهدوء (status: available=false مع عرض الإصدار الحالي).
- اختُبر: المحدّث بمحاكاة كاملة (check→downloaded→install)، الإصدار، الحماية بالمصادقة، والتعطيل الآمن بدون electron-updater (6/6 + تحقق standalone).

**مراقبة إشارة المكسر (`core/signal.js` + `panel/js/signal.js` + تكامل الشاشة)** — مُختبَرة end-to-end:
- إشارة كهربائية (نشط/متوقف) لا صوت. طريقان: عبر الشبكة (POST /api/signal و /ping بنمط نبض مع watchdog)، أو عبر منفذ تسلسلي (استخراج بِت الحالة: activeByteIndex/activeMask/activeHigh، مع طلب hex دوري اختياري).
- تبثّ الحالة للشاشة فوراً عبر WebSocket (`{type:'signal'}`). الشاشة في وضع الإقامة: تبقى أثناء نشاط الإشارة، وتعود للأذكار بعد انقطاعها لمدة `iqamaMode.returnSilenceSec`؛ ومع تعطيل المراقبة تعمل بالمؤقّت الثابت (احتياطي آمن).
- نقاط الإشارة مفتوحة على الشبكة المحلية ليرسلها جهاز خارجي بلا مصادقة؛ الحالة (status) تتطلب مصادقة.
- اختُبر: منطق المراقبة وقراءة التسلسلي (12/12)، المسارات + البث (6/6)، وتكامل العودة في الشاشة (6/6).

**جهاز GPS (`core/nmea.js` + `core/gps.js` + `panel/js/gps.js`)** — اختياري ومُختبَر منطقياً:
- كشف تلقائي للمنفذ (يفحص المنافذ ويختار ما يبثّ جُمل NMEA صحيحة)، مع دعم اختيار منفذ يدوي.
- محلّل NMEA كامل (RMC/GGA، جميع معرّفات المتحدث، تحقّق checksum، تحويل الإحداثيات، استخراج وقت UTC مع محور سنتين) — مُختبَر بأمثلة حقيقية.
- عند الإشارة: ضبط `location` تلقائياً (مع عتبة تحرّك >30م)، وضبط الوقت عبر `timeSync.setExternal` بمصدر «GPS» — كله دون إنترنت.
- مكتبة `serialport` ضمن `optionalDependencies`؛ النظام يعمل بأمان لو لم تُثبَّت (الميزة تُعطَّل بهدوء).
- مسارات `GET /api/gps/status`، `GET /api/gps/ports`، `POST /api/gps/scan`. اختُبر منطق الاستيعاب والموقع/الوقت (10/10) والمسارات (7/7).
- لم يُختبَر فتح المنفذ التسلسلي فعلياً (يحتاج عتاد GPS)؛ منطق التحليل والقرار معزول ومُختبَر بالكامل.

**الشاشة (`display/`)** — مبنية:
- 4 أوضاع: الرئيسية، العدّاد الدائري، الإقامة (شاشة سوداء)، الأذكار.
- العدّاد الدائري (دقائق في الوسط، يتحول لثوانٍ في الدقيقة الأخيرة حتى الصفر).
- نغمة تنبيه قصيرة عند دخول الوقت (WebAudio، قابلة للتعطيل والتحكم بالصوت).
- آلة حالة تلقائية: نافذة [الأذان←الإقامة] تُظهر العدّاد، ثم شاشة الإقامة، ثم الأذكار، ثم العودة للرئيسية.
- أوضاع أذكار: شاشة/شاشتان/ثلاث/كل ذكر مستقل. بانر الجنازة. ثلث الليل. أفقي + عمودي.

**لوحة التحكم (`panel/`)** — مبنية:
- تبويبات: الهوية، الأوقات، الإقامة، التحكم الحي، الأذكار، الجنازة، المناسبات، المستخدمون، السجل، النظام.
